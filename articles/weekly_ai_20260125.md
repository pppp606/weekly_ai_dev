## 🚀 リリース情報

### Claude Code 2.1.19

Claude Codeは今週もアクティブに更新されています。

- **タスクシステム環境変数**: `CLAUDE_CODE_ENABLE_TASKS`環境変数で新しいタスクシステムを一時的に無効化可能に
- **コマンド引数ショートハンド**: カスタムコマンドで`$0`、`$1`などで個別引数にアクセス可能に
- **スキル変更**: 追加権限やフックを持たないスキルは承認なしで実行可能に
- **VSCode**: セッションフォークと巻き戻し機能が全ユーザーに開放

バージョン2.1.18では**カスタマイズ可能なキーボードショートカット**が追加され、コンテキストごとのキーバインディング設定やコードシーケンスの作成が可能になりました。

https://github.com/anthropics/claude-code

### Cursor 2.4 - Subagents, Skills, and Image Generation

Cursorは2.4で大型アップデートを実施しました。

- **サブエージェント**: 独立したエージェントが離散的なタスクを並列処理。カスタムプロンプトとツールアクセスを持ち、専用のコンテキストウィンドウで動作
- **スキル**: `SKILL.md`ファイルでドメイン特化の機能を定義。プロジェクトディレクトリに保存してチームで共有可能
- **画像生成**: テキスト説明からエージェント内で直接画像を生成。参照画像のアップロードにも対応
- **Cursor Blame（Enterprise）**: コード行をAI生成の会話サマリーにリンクし、AI支援によるコード変更の監査証跡を提供
- **Agent Q&A**: エージェントがバックグラウンドで作業を続けながら、明確化のための質問が可能に

CLIでは**Plan Mode**（`/plan`または`--mode=plan`）と**Ask Mode**（`/ask`または`--mode=ask`）が追加され、ワード単位のdiff表示やMCP認証のワンクリックログインも実装されました。

https://cursor.com/changelog

### GitHub Copilot CLI 0.0.394

- **コンテキスト重複排除**: 同一のモデル指示ファイルを重複排除してコンテキストを節約
- **GHE Cloudサポート**: `/delegate`コマンドでGitHub Enterprise Cloud（*.ghe.com）をサポート
- **Esc-Esc Undo**: Esc-Escショートカットでファイル変更を任意の以前のスナップショットに戻す機能（0.0.393）
- **プラグインコマンド**: `/plugin`コマンドでプラグインマーケットプレイス管理が可能に（0.0.392）

https://github.com/github/copilot-cli

### Kiro CLI 1.24.0

- **スキルシステム**: 大規模ドキュメントセットの段階的コンテキストロード、タスク要件に基づく動的コンテキスト発見
- **カスタムDiffツール**: deltaやdifftasticなどの外部diffオプションをサポート
- **ASTパターンツール**: `pattern-search`でASTクエリによるコードパターン検索、`pattern-rewrite`でASTトランスフォーメーションによる精密なリファクタリング。追加のLSP設定なしで18言語に対応
- **/compactコマンド**: 会話履歴を管理してコンテキスト効率を維持

https://kiro.dev/changelog/

### OpenAI Codex 0.89.0

https://github.com/openai/codex

## 📈 注目のAI開発リポジトリ

### AionUi - 複数CLIベースAIツールの統合デスクトップアプリ

https://github.com/iOfficeAI/AionUi

Claude Code、Gemini CLI、Codex、OpenCodeなど、ターミナルで動くAIコーディングエージェントを一つのGUIで統合管理できるオープンソースアプリケーションです。

- **マルチエージェント統合**: 主要なCLIツールを自動検出して一元管理
- **独立セッション管理**: 各会話は独立したコンテキストを持ち、SQLiteでローカル保存
- **マルチフォーマットプレビュー**: PDF、Word、Excel、PowerPoint、コード、Markdownなど9種類以上のファイルをアプリ内でプレビュー可能
- **WebUIモード**: ネットワーク越しに他のデバイスからアクセスできるリモートモード搭載

AIコーディングツールの選択肢が増えた結果、「どのツールをどの場面で使うか」という新しい課題が生まれています。AionUiはGUI化によってツール切り替えのオーバーヘッドを減らし、全ての会話をローカルのSQLiteに保存することで履歴の一元管理も実現しています。

### PageIndex - ベクトルDBを使わないRAG

https://github.com/VectifyAI/PageIndex

PageIndexは「ベクトルDBを使わないRAG」です。文書をチャンキングしてベクトル化する代わりに、目次のような階層的なインデックス構造を自動生成し、LLMの推論能力でツリーを辿りながら情報を検索します。

- **ベクトルDB不要**: 埋め込みモデルもベクトルストアも不要。シンプルなアーキテクチャで導入障壁が低い
- **チャンキング不要**: 文書の自然な構造を保持し、人工的な分割による情報の断絶を回避
- **透明な結果**: ページ参照付きの追跡可能な結果。「なぜその回答に至ったか」が分かる
- **MCP統合**: Model Context Protocolに対応

FinanceBenchという金融QAベンチマークで98.7%の精度を達成。説明責任が求められる領域での活用が期待されます。

## 📄 今週のAI論文トレンド

### 1. EvoCUA: Evolving Computer Use Agents via Learning from Scalable Synthetic Experience

**著者**: Taofeng Xue, Chong Peng, Mianqiu Huang 他

ネイティブなコンピュータ使用エージェント（CUA）の開発において、静的データスケーリングの制約を克服するEvoCUAという進化的アプローチを提案。検証可能な合成エンジンで多様なタスクを自律的に生成し、反復的進化学習戦略によって能力境界を特定しながら、失敗軌跡を監督信号へと変換します。OSWorldベンチマークで56.7%の成功率を達成し、オープンソースの新たな最先端記録を樹立。

https://arxiv.org/abs/2601.15876

### 2. HERMES: KV Cache as Hierarchical Memory for Efficient Streaming Video Understanding

**著者**: Haowei Zhang, Shudong Yang, Jinlan Fu 他

ストリーミング動画入力への拡張課題に対処するため、HERMESというトレーニング不要のアーキテクチャを提案。KVキャッシュを複数の粒度にわたって動画情報をカプセル化する階層的メモリフレームワークとして概念化しています。従来のSOTAと比較してTTFT（Time To First Token）が10倍高速化され、動画トークンを最大68%削減しながらストリーミングデータセットで最大11.4%の性能向上を達成。

https://arxiv.org/abs/2601.14724

### 3. LLM-in-Sandbox Elicits General Agentic Intelligence

**著者**: Daixuan Cheng, Shaohan Huang, Yuxian Gu 他

LLMがコードサンドボックス（仮想コンピュータ）内で探索することで、非コード領域における汎用知能を引き出すLLM-in-Sandboxを提案。LLMは自発的に外部リソースにアクセスして新しい知識を獲得し、ファイルシステムを活用して長いコンテキストを処理し、フォーマット要件を満たすためにスクリプトを実行します。数学、物理、化学、生物医学、長文脈理解、指示追従にわたる堅牢な汎化を達成。

https://arxiv.org/abs/2601.16206
