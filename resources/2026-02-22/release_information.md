## 1. Claude Code (Anthropic)

- **リポジトリ**: https://github.com/anthropics/claude-code
- **CHANGELOG**: https://github.com/anthropics/claude-code/blob/main/CHANGELOG.md
- **最新バージョン**: 2.1.50（2026-02-20更新）

### バージョン 2.1.50 の主な変更点

#### 新機能
- **Worktreeベースのエージェント分離**: エージェント定義に `isolation: worktree` を指定することで、エージェントが独立したgit worktreeで実行可能に。開発ワークフローで並行作業が安全に行える
  ```yaml
  # .claude/agents/reviewer.md のフロントマターに追加
  isolation: worktree
  ```
- **WorktreeCreate/WorktreeRemove フックイベント**: worktree作成・削除時にカスタムVCSセットアップやティアダウンスクリプトを実行可能
- **`claude agents` CLIコマンド**: 設定済みの全エージェントを一覧表示する新コマンド
- **LSPサーバーの`startupTimeout`設定サポート**: LSPサーバーの起動タイムアウトを設定可能に
- **`CLAUDE_CODE_DISABLE_1M_CONTEXT`環境変数**: 1Mコンテキストウィンドウサポートを無効化するオプション
- **Opus 4.6（ファストモード）で1Mコンテキストウィンドウをフルサポート**
- **VSCode: `/extra-usage`コマンドサポート追加**

#### メモリ・パフォーマンス改善（多数）
- エージェントチームで完了したタスクがガベージコレクションされないメモリリークを修正
- 完了タスクの状態オブジェクトがAppStateから削除されないメモリリークを修正
- LSP診断データが配信後にクリーンアップされないメモリリークを修正
- TaskOutputがクリーンアップ後にrecent linesを保持するメモリリークを修正
- CircularBufferでクリアされたアイテムがバッキング配列に残るメモリリークを修正
- シェルコマンド実行でChildProcessとAbortControllerの参照が保持されるメモリリークを修正
- 長時間セッション中の内部キャッシュクリア、大きなツール結果の処理後クリア
- ファイル履歴スナップショットの上限設定による無制限メモリ増加の修正
- ヘッドレスモード（`-p`フラグ）の起動パフォーマンス改善

#### `CLAUDE_CODE_SIMPLE`モードの強化
- MCP tools、添付ファイル、フック、CLAUDE.mdファイルの読み込みも無効化し、完全にミニマルな体験を提供
- skills、セッションメモリ、カスタムエージェント、CLAUDE.mdトークンカウントを完全にストリップ

### バージョン 2.1.49 の主な変更点（期間内）

#### 新機能
- **`--worktree`（`-w`）フラグ**: 分離されたgit worktreeでClaudeを起動
- **サブエージェントの`isolation: "worktree"`サポート**: 一時的なgit worktreeでの作業が可能
- **Ctrl+Fキーバインドでバックグラウンドエージェントを終了**（2回押しで確認）
- **エージェント定義で`background: true`サポート**: 常にバックグラウンドタスクとして実行
- **プラグインが`settings.json`をデフォルト設定として同梱可能**
- **`ConfigChange`フックイベント**: 設定ファイル変更時に発火し、エンタープライズセキュリティ監査やブロッキングが可能
- **MCP OAuth認証の改善**: ステップアップ認証サポートとディスカバリキャッシング
- **SDKにモデル機能検出フィールド追加**: `supportsEffort`、`supportedEffortLevels`、`supportsAdaptiveThinking`

#### モデル変更
- **Sonnet 4.5（1Mコンテキスト）がMaxプランから削除** → Sonnet 4.6（1Mコンテキスト）への移行を推奨

#### パフォーマンス改善
- tree-sitterパーサーの定期的リセットによるWASMメモリリーク修正
- MCP認証失敗のキャッシュによる起動パフォーマンス改善
- アナリティクストークンカウントのHTTPコール削減
- MCPツールトークンカウントのバッチ化

- AnthropicのClaude Agent SDKをVS Code内で直接利用可能。GitHub Copilotサブスクリプションに含まれるClaudeモデルを使用

## 6. Kiro IDE

- **ウェブサイト**: https://kiro.dev
- **Changelog**: https://kiro.dev/changelog/
- **最新リリース**: Version 0.10（2026-02-18）

### Version 0.10 の主な変更点

#### 設計ファーストの機能仕様（Design-First Feature Specs）
- 要件からではなく技術アーキテクチャから機能仕様を開始。高レベルの設計、擬似コード、システム図を入力すると、実現可能な要件を自動的に導出
- 厳密な非機能要件やポーティング対象の既存設計を持つプロジェクトに適している

#### バグ修正仕様（Bugfix Specs）
- 根本原因分析、修正設計、回帰防止をガイドする専用ワークフロー
- 現在の動作、期待される動作、変更してはいけない部分を記録し、コード生成前の明確なガードレールを提供

#### Hunkベースレビュー（Supervised Mode）
- ファイル変更が全体差分ではなく個別の論理的Hunkとして表示
- 各Hunkを独立して承認、拒否、またはインラインチャットで議論可能
- ファイル単位やプロジェクト単位での変更承認もサポート

#### Pre/Postタスク実行フック
- **Pre-executionフック**: タスク開始前にセットアップスクリプトの実行や前提条件の検証
- **Post-executionフック**: タスク完了後にテスト、リンティング、外部システム通知をトリガー

#### MCPプロンプトとリソーステンプレート
- MCPサーバーがプロンプトとリソーステンプレートをコンテキストプロバイダーメニュー経由で提供
- テンプレートパラメータを入力して特定のコンテンツを取得
- ツール実行中にサーバーが追加のユーザー入力を要求（elicitation）可能

#### パッチ
- **0.10.16**（2月20日）: スペックタスク実行中のMCPツール呼び出し問題を修正
- **0.10.10**（2月20日）: Linuxでの`.kiro/`パス解決、新セッションでのメッセージ表示、フックでの`$USER_PROMPT`環境変数の修正
