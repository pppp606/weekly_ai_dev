# リリース情報まとめ (2026-01-25 〜 2026-02-01)

## 1. Gemini CLI v0.26.0

- **リリース日**: 2026-01-28
- **リポジトリ**: https://github.com/google-gemini/gemini-cli
- **リリースページ**: https://github.com/google-gemini/gemini-cli/releases/tag/v0.26.0

### 主な新機能・変更点

#### Agent Skills がデフォルトで有効化
- エージェントスキル機能がデフォルトで有効になり、サブエージェントによるタスク委任がすぐに利用可能に
- 新しい `skill-creator` ビルトインスキルが導入され、カスタムスキルの作成とCJSパッケージ管理が可能に
- スキルのスコープが `project` から `workspace` に変更

#### Plan モード（実験的機能）
- `--plan` フラグによる実験的なプランモードが追加
- プランモードでは厳格な読み取り専用ポリシーが適用され、違反時に実行を停止
- `read_many_files` が承認モードポリシーから除外され、より柔軟な計画立案が可能

#### Generalist Agent の追加
- 汎用的なタスクを処理する `generalist` エージェントが新たに追加
- 外部エージェント(expert subagent)が存在する場合、メインエージェントがそれらを活用するよう誘導

#### `/introspect` スラッシュコマンド
- 現在のセッション状態やエージェント構成を確認できる新しいデバッグコマンド

#### セキュリティとスキル管理の強化
- スキルインストール時のセキュリティ同意プロンプトが追加
- スキルのオーバーライドに対する競合検出と警告機能
- シェルリダイレクトの透過性とセキュリティが改善

#### コンテキスト圧縮の改善
- アンカー付き反復コンテキスト圧縮と自己検証機能が強化
- コンテキストオーバーフローチェック前に圧縮を試みるよう改善

#### その他の改善
- `--prompt` フラグの非推奨が解除
- `auto` エイリアスによるデフォルトモデル選択が追加
- 設定名が `disable*` から `enable*` のポジティブな命名に変更
- MCP サーバーのロード前にコマンド実行が可能に
- Esc-Esc でプロンプトクリアが可能に
- Windows クラッシュの修正（PTY リサイズ問題）
- OOM クラッシュ防止のためのファイル検索走査制限

## 2. Claude Code (v2.1.20 〜 v2.1.29)

- **更新日**: 2026-01-27 〜 2026-01-31（複数バージョンリリース）
- **リポジトリ**: https://github.com/anthropics/claude-code
- **CHANGELOG**: https://github.com/anthropics/claude-code/blob/main/CHANGELOG.md

### 主な新機能・変更点

#### v2.1.29
- セッション再開時の `saved_hook_context` によるスタートアップパフォーマンス問題を修正

#### v2.1.27 - PR連携の強化
- `--from-pr` フラグが追加され、特定のGitHub PR番号やURLに紐づくセッションの再開が可能に
  ```bash
  claude --from-pr 123
  claude --from-pr https://github.com/org/repo/pull/123
  ```
- `gh pr create` でPR作成時にセッションが自動的にPRにリンクされるように
- ツールコール失敗・拒否がデバッグログに記録されるように
- Windowsでの `.bashrc` 関連のbash実行失敗を修正
- VSCode: OAuth トークン期限切れによる401エラーを修正

#### v2.1.25
- Bedrock/Vertex のゲートウェイユーザー向けベータヘッダーバリデーションエラーを修正

#### v2.1.23 - パフォーマンスとプロキシ改善
- カスタマイズ可能なスピナー動詞設定 (`spinnerVerbs`) が追加
- mTLS とプロキシ接続性の修正（企業プロキシやクライアント証明書利用者向け）
- プロンプトキャッシュスコープ有効時の400エラーを引き起こすレースコンディションを修正
- ターミナルレンダリングパフォーマンスの最適化
- Bash コマンドでタイムアウト時間が経過時間と共に表示されるように

#### v2.1.22
- 非対話モード (-p) での構造化出力を修正

#### v2.1.21 - 日本語IME対応とVSCode改善
- 日本語IMEの全角数字入力に対応（オプション選択プロンプト）
- Claudeがファイル操作ツール（Read, Edit, Write）をbash相当コマンド（cat, sed, awk）より優先するよう改善
- [VSCode] Python仮想環境の自動アクティベーション機能が追加
  - `claudeCode.usePythonEnvironment` 設定で制御可能
- セッション中断後の再開時のAPIエラーを修正
- 大きな出力トークン制限を持つモデルでの自動コンパクト早期トリガーを修正

#### v2.1.20 - PR レビューステータスとタスク管理
- PR レビューステータスインジケーターがプロンプトフッターに追加（承認済み、変更要求、保留中、ドラフト状態を色付きドットで表示）
- `--add-dir` フラグで追加ディレクトリから `CLAUDE.md` ファイルのロードをサポート
- `TaskUpdate` ツールによるタスク削除機能が追加
- セッションコンパクション問題の修正（再開時にフルヒストリーがロードされる問題）
- エージェントが作業中にユーザーメッセージを無視する問題を修正
- ワイド文字（絵文字、CJK）のレンダリング問題を修正
- `/commit-push-pr` スキルがMCPツール経由でSlackチャンネルにPR URLを自動投稿するように
- `/copy` コマンドが全ユーザーに開放

- **リリース日**: 2026-01-30
- **リポジトリ**: https://github.com/cline/cline
- **リリースページ**: https://github.com/cline/cline/releases/tag/v3.56.2

### 主な新機能・変更点

#### CLI認証の拡張
- Vercel AI Gateway と Cline APIキープロバイダーがヘッドレスCI/自動化ワークフロー向けにサポート

- Moonshot プロバイダーに **Kimi-K2.5** モデルを追加

#### Hooks の改善
- フックスクリプトがファイルシステムルートではなくワークスペースリポジトリルートから実行されるように変更

#### デフォルト設定の変更
- マルチルートワークスペース、並列ツール呼び出し、スキルがデフォルトで有効に
- ストリクトプランモードがデフォルトで無効に

#### OpenTelemetry の改善
- メトリクスとログエンドポイントにカスタムヘッダーのサポートを追加
- エンドポイントパス処理の修正（不正な `/v1/logs` や `/v1/metrics` の追加を防止）
- VSCode テレメトリ設定に関わらずログ送信を保証

#### その他
- Trinity Large プロンプトバリアントの追加（ツール呼び出しサポートの改善）
- LiteLLM での推論対応モデルのthinking設定表示問題を修正

## 4. OpenAI Codex v0.91.0 〜 v0.93.0

- **リリース日**: 2026-01-25 〜 2026-01-31（3バージョンリリース）
- **リポジトリ**: https://github.com/openai/codex
- **最新リリースページ**: https://github.com/openai/codex/releases/tag/rust-v0.93.0

### v0.93.0 (2026-01-31) の主な変更点

#### Plan モードの強化
- 提案されたプランを専用TUIビューにストリーミング表示
- `/plan` ショートカットによるクイックモード切替（フィーチャーゲート付き）

#### Smart Approvals がデフォルト有効に
- スマート承認機能がデフォルトで有効化
- MCP ツール呼び出しに対する明示的な承認プロンプトが追加

#### `/apps` コネクタブラウザ
- TUI 内でコネクタを閲覧する `/apps` コマンドと、アプリプロンプト用の `$` 挿入機能が追加

#### SQLite ベースのログデータベース
- SQLite をバックエンドにしたログDBが導入
- スレッドIDフィルタリング、保持期間管理、ヒューリスティックカラーリング機能

#### SOCKS5 プロキシサポート
- ポリシー強制と設定ゲーティング付きのオプションSOCKS5プロキシリスナーが追加

#### 外部認証モード
- App-server が外部認証モードをサポートし、ホストアプリからのChatGPT認証トークンを受け入れ可能に

### v0.92.0 (2026-01-27) の主な変更点

#### マルチエージェントコラボレーション強化
- エクスプローラーロール、コラボイベントマッピング改善、最大深度ガードレールの追加
- サブエージェント待機時のCPU高負荷問題を修正

#### 動的ツール注入
- API v2 スレッドで起動時に動的ツールを注入し、サーバーとコアツールパイプラインを通じてルーティング可能に

#### セッション管理の改善
- `thread/unarchive` RPC でアーカイブ済みセッションの復元が可能に
- スレッドリストのフィルタリング機能が追加

#### MCP OAuth スコープ設定
- `config.toml` で OAuth スコープを定義可能に（ログイン毎の `--scopes` 指定が不要に）

### v0.91.0 (2026-01-25) の変更点
- サブエージェントの最大数を6に削減し、リソース使用量とガードレールを強化

## 5. GitHub Copilot CLI v0.0.400

- **リリース日**: 2026-01-30
- **リポジトリ**: https://github.com/github/copilot-cli
- **リリースページ**: https://github.com/github/copilot-cli/releases/tag/v0.0.400

### 主な新機能・変更点

#### Autopilot モード（実験的）
- 自律的なタスク完了のための Autopilot モードが実験的に追加

#### MCP サーバー管理の改善
- MCP サーバーのインストラクションサポートが追加
- シャットダウン時にMCPサーバーへDELETEリクエストを送信
- リモートMCPサーバーのOAuth互換性が改善

#### テーマとUI
- `/theme` コマンドと GitHub Dark/Light テーマによるテーマピッカーが追加
- 順序付きリストがダッシュではなく番号で表示されるように
- Markdown テーブルヘッダーが太字で表示
- 思考中のターミナルタブにプログレスインジケーターを表示

#### プラグイン管理
- `copilot plugin` サブコマンドによる非対話型プラグイン管理が追加

#### パフォーマンスと操作性改善
- 多くのメッセージがあるセッションでのレスポンシブ性が向上
- UNIX キーボードバインディング（Ctrl+A/E/W/U/K、Alt+矢印キー）のサポート改善
- grep/glob ツールの大量結果によるメモリ問題への対処
- バックスペースの長押し削除に対応
- Diff モードのファイルリストがカルーセルナビゲーションに（最大5ファイル表示）
- `launch_messages` 設定によるスタートアップアナウンスメント

#### その他
- `ask_user` ツールプロンプトへのユーザー応答がタイムラインにユーザー名付きで表示
- Windows Terminal での大量コンテンツ貼り付けをサポート
- バンドルされたLSPサーバー（TypeScript、Python）の削除
- Code Review ツールが大規模な変更セットを処理（ビルドアーティファクト除外、100ファイル制限）
