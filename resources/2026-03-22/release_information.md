## 1. Gemini CLI v0.34.0

### 主な新機能・変更点

#### Plan Modeがデフォルトで有効化
- `Plan Mode`がデフォルトで有効になり、実験的バッジが削除された
- ドキュメントにも反映され、Conductorサンプルを使った計画ワークフローの説明が追加

- `/compact`を`/compress`のエイリアスとして追加
- `/chat`と`/resume`コマンドのUXを統一
- `/upgrade`コマンドの実装
- `/tools desc`をサブコマンドとして公開
- スキルのスラッシュコマンドによる有効化機能
- スラッシュコマンド衝突時のドットプレフィックス実装

#### パフォーマンス・安定性
- **OOMクラッシュ修正**: 長時間セッションでのメモリ不足クラッシュを修正

- **キーチェーン統合**: KeychainServiceを統一し、トークンストレージを移行
- **CacheService実装**: 汎用CacheServiceを実装し、setupUserを最適化
- **ループ検出**: 反復的なループ検出とモデルフィードバック機構を実装

- A2Aの正規化とストリーミング再組み立てを標準化
- ACPフラグを`--experimental-acp`から`--acp`にリネーム
- AI Gatewayの認証サポートを追加

#### セキュリティ・ポリシー
- サブエージェント固有のポリシーをTOMLでサポート
- IPバリデーションとsafeFetch基盤を実装

#### その他

## 2. Claude Code v2.1.81

- **更新日**: 2026-03-20
- **リポジトリ**: https://github.com/anthropics/claude-code
- **CHANGELOG**: https://github.com/anthropics/claude-code/blob/main/CHANGELOG.md

### v2.1.81の主な変更点

#### 新しい`--bare`フラグ
- スクリプト用の`-p`呼び出し向けに`--bare`フラグを追加
- フック、LSP、プラグイン同期、スキルディレクトリウォークをスキップ
- `ANTHROPIC_API_KEY`または`--settings`経由の`apiKeyHelper`が必要（OAuth/キーチェーン認証は無効）
- 自動メモリも完全無効化

```bash
# 使用例: スクリプトからの軽量呼び出し
claude -p --bare "コードを生成して"
```

#### `--channels`パーミッションリレー
- チャンネルサーバーがパーミッション機能を宣言している場合、ツール承認プロンプトをスマートフォンに転送可能

#### バグ修正
- 複数のClaude Codeセッションが同時実行時にOAuthトークンリフレッシュで再認証を要求する問題を修正
- ボイスモードのリトライ失敗が静かに飲み込まれる問題を修正
- ボイスモードのWebSocket接続断からのオーディオ回復問題を修正
- `CLAUDE_CODE_DISABLE_EXPERIMENTAL_BETAS`が構造化出力ベータヘッダーを抑制しない問題を修正
- Node.js 18でのクラッシュを修正
- ダッシュを含むBashコマンドの不要なパーミッションプロンプトを修正
- バックグラウンドエージェントタスクの出力がハングする競合状態を修正

#### UX改善
- MCP read/searchツールの呼び出しが「Queried {server}」1行に折りたたまれるように改善（Ctrl+Oで展開）
- `!` bashモードの発見性を向上
- プラグインの更新性を向上（ref-trackedプラグインがロードごとに再クローン）
- Remote Controlセッションタイトルが3メッセージ後にリフレッシュ

#### その他の変更
- MCP OAuthでCIMD (SEP-991)をサポート
- プランモードの「コンテキストクリア」オプションをデフォルトで非表示に
- Windows（WSL含む）でのライン単位ストリーミングを無効化
- [VSCode] Windows PATHの継承問題を修正（Git Bash使用時）

### v2.1.80の主な変更点（同週内リリース）

#### 修正
- `--resume`が並行ツール結果をドロップする問題を修正
- ボイスモードWebSocketのCloudflareボット検出対応
- 大規模リポジトリでの`@`ファイルオートコンプリートの応答性改善

## 3. Cursor - Composer 2

- **リリース日**: 2026-03-19
- **ウェブサイト**: https://cursor.com
- **Changelogページ**: https://cursor.com/changelog

### 概要

CursorがComposer 2を発表。フロンティアレベルのコーディングパフォーマンスを実現する新しいAIモデル。

### ベンチマーク結果

| ベンチマーク | Composer 2 | Composer 1.5 | Composer 1 |
|-----------|-----------|-------------|-----------|
| CursorBench | 61.3 | 44.2 | 38.0 |
| Terminal-Bench 2.0 | 61.7 | 47.9 | 40.0 |
| SWE-bench Multilingual | 73.7 | 65.9 | 56.9 |

Composer 2はすべてのベンチマークで大幅な改善を達成。

### 技術的アーキテクチャ

1. **Continued Pretraining**: 基礎モデルを強化する拡張事前学習フェーズ
2. **強化学習**: 長期的なコーディングタスクでのトレーニング。数百のアクションを必要とする複雑なタスクの解決が可能

### 価格体系

- **Standard**: 入力$0.50/M、出力$2.50/Mトークン
- **Fast（デフォルト）**: 入力$1.50/M、出力$7.50/Mトークン

## 4. VS Code 1.112.0（AI関連機能のみ）

- **リリース日**: 2026-03-18
- **リポジトリ**: https://github.com/microsoft/vscode
- **リリースページ**: https://github.com/microsoft/vscode/releases/tag/1.112.0
- **更新情報**: https://code.visualstudio.com/updates/v1_112

### AI/Copilot関連の新機能

#### エージェント体験の改善
- **メッセージステアリングとキューイング**: Copilot CLIでリクエスト実行中にメッセージを送信可能。エージェントの方向転換やフォローアップメッセージのキューイングに対応
- **委任前のプレビュー変更**: Copilot CLIにタスクを委任する際にChat viewで保留中の変更を表示
- **Copilot CLIターミナルのクリック可能リンク**: ファイルパスが機能的なリンクとして認識される
- **パーミッションレベル**: 3段階の自律性設定を追加
  - Default Permissions（承認が必要）
  - Bypass Approvals（自動承認）
  - Autopilot（自動承認＋質問への自動応答）
- **`/troubleshoot`スキル**: エージェントのデバッグログを会話内で分析して問題を診断
- **デバッグログのエクスポート/インポート**: セッションのエージェント動作データをオフラインで共有・分析可能

#### エージェントの機能拡張
- **画像・バイナリファイルサポート**: エージェントが画像ファイルやバイナリデータ（hexdump表示）を読み取り可能。スクリーンショット分析やダイアグラム解釈に対応
- **自動シンボル参照**: シンボルをチャットにコピー＆ペーストすると自動的にシンボル参照としてフォーマット

#### エージェント拡張性
- **親リポジトリからのカスタマイズ検出**: モノレポ環境でエージェントの指示やツールをパッケージ間で共有可能
- **MCPサーバーサンドボックス**: macOSとLinuxでローカルMCPサーバーを制限された環境で実行可能
- **プラグイン/MCPサーバーの有効/無効切り替え**: アンインストールなしでグローバルおよびワークスペース単位で切り替え可能
- **自動プラグインアップデート**: 設定に基づいてプラグインが自動更新
