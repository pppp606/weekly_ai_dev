### 3. [AIエージェント（Cursor/Claude Code他）への「指示だし」レベルを上げる教育プロンプト](https://qiita.com/WdknWdkn/items/762e9020c9f4e84e3c51)
- **著者**: WdknWdkn (Kentaro Wada) - ierabu GROUP 執行役員
- **プラットフォーム**: Qiita
- **公開日**: 2025-09-15
- **いいね数**: 高評価
- **概要**: Cursor や Claude Code などの AI コーディングエージェントに対する指示の質を評価・改善するための教育フレームワークを提案。経験豊富な開発者は AI を効果的に活用できる一方、多くのチームが曖昧な指示により非効率な結果に直面している現状を改善します。
- **開発者向けポイント**:
  - **目的の明確化 (Done Definition)**: 「認証を実装して」ではなく、成功条件とテストコマンドを含む具体的な完了基準を提示
  - **制約の明示**: 技術的・ビジネス的制約を明示することで、AI の探索空間を絞り、望ましくない実装を防止
  - **コンテキスト共有**: Cursor の `@file`、`@folder`、`@Symbols` 機能を活用して、正確なファイル参照、入出力例、実行ログを提供
  - **段階的進行**: 設計 → スケルトンコード → コアロジック → エラーハンドリング → テスト、という順序で複雑なタスクを分解
  - **成熟度レベル (L1-L5)**: 指示を L1（曖昧な指示、最小限のコンテキスト）から L5（包括的なコンテキスト、ツールマスタリー、プロジェクト全体の認識）まで段階的に評価
- **実装例**: 目的、制約、コンテキスト、ステップバイステップの方法論、検証手順を組み合わせた再利用可能なプロンプトテンプレートを提供

## トレンドトピック

### AI開発ツールの選択基準の変化
2025年、AI開発ツールの選択基準が大きく変化しています。以前は「Claude Code + Sonnet 4.5」が主流でしたが、11月時点では「Cursor × GPT-5-codex high」が最適解として注目されています。この変化は、AI の急速な進化により、数ヶ月単位でツールの優位性が変わることを示しています。

### ターミナルベースのAI開発環境の台頭
Cursor CLI Beta の登場により、IDE に依存しない AI 開発支援が現実のものとなりました。これにより、開発者は使い慣れた環境を維持しながら、最新の AI 支援機能を活用できるようになっています。

### AI への「指示力」の重要性
単に AI ツールを導入するだけでなく、AI に対してどのように指示を出すかが開発効率に直結することが認識されつつあります。L1〜L5 の成熟度レベルのような指標を用いて、チーム全体の「AI 活用力」を底上げする取り組みが始まっています。

### マルチモデル対応の重要性
単一のLLMに依存するのではなく、タスクに応じて最適なモデルを選択できるツール（Cursor CLI のようなマルチモデル対応）が求められています。コスト、精度、速度のバランスを考慮した柔軟なモデル選択が、実践的なAI開発の鍵となっています。

### Windows環境でのAI開発環境整備
WSL を活用した Windows 環境での AI 開発が一般化しつつあります。これにより、Windows ユーザーも Linux ベースのツールをネイティブに近い形で利用でき、開発環境の選択肢が広がっています。
