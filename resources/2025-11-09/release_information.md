### Google Gemini CLI v0.15.0-nightly (2025-11-07)

Google Gemini CLIの最新ナイトリービルドがリリースされました。

**主な変更点:**
- **スクロール可能なサポート**: ターミナルUIの改善により、より多くの情報を扱いやすくなりました
- **サブエージェントへの圧縮機構追加**: サブエージェントでのコンテキスト管理が改善され、より効率的な処理が可能になりました
- **拡張機能のリロード機能**: カスタムコマンドの更新や有効/無効化のコマンドが追加されました
- **セキュリティ強化**: ポリシーエンジンでのサーバー名スプーフィング防止機能が追加されました
- **モデル設定サービスの追加**: `ModelConfigService`により、モデル設定の管理が改善されました
- **シェルコマンド説明の改善**: シェルコマンドの説明に現在のワーキングディレクトリ(cwd)が含まれるようになりました
- **エラーハンドリングの改善**: Flash モデルのエラーやクォータ/キャパシティメッセージの処理が改善されました

**開発への影響:**
- より安定したサブエージェント機能により、複雑なタスクの自動化がさらに進化
- ポリシーエンジンの強化により、エンタープライズ環境でのセキュリティが向上
- 拡張機能のリロード機能により、開発体験が向上

### Claude Code v2.0.36 (2025-11-07 CHANGELOG更新)

Anthropic の Claude Code CHANGELOG が更新され、複数の新機能とバグ修正が追加されました。

**v2.0.36の主な変更点:**
- **自動更新通知の制御**: `DISABLE_AUTOUPDATER` 環境変数でパッケージマネージャーの更新通知を適切に無効化できるようになりました
- **キューメッセージの処理改善**: キューに入れられたメッセージが誤ってbashコマンドとして実行される問題を修正
- **入力の保持**: キューメッセージ処理中の入力喪失問題を修正

**v2.0.35の主な変更点:**
- **コマンド検索の改善**: ファジー検索結果の精度が向上
- **自動終了機能**: `CLAUDE_CODE_EXIT_AFTER_STOP_DELAY` 環境変数により、SDKモードでの自動終了が可能に（自動化ワークフローやスクリプトに有用）
- **設定の移行**: `ignorePatterns` がプロジェクト設定からローカル設定の拒否権限に移行
- **VS Code統合の改善**: `chat.fontSize` と `chat.fontFamily` の設定をUI全体で尊重し、リロード不要で即座に適用

**開発への影響:**
- 自動化ワークフローでの使いやすさが大幅に向上
- VS Code での視覚的な一貫性とユーザビリティが改善
- メッセージ処理の信頼性向上により、より安定した対話が可能

**v2.0.34以前の注目機能:**
- **Plan Mode改善**: 新しいPlanサブエージェントの導入
- **Plugin System**: カスタムコマンド、エージェント、フック、MCPサーバーでの拡張性
- **Explore subagent**: Haikuを活用した効率的なコードベース検索
- **Haiku 4.5サポート**: Pro ユーザー向けに Haiku 4.5 が利用可能に
- **Sandbox Mode**: Linux & Mac での BashTool 用サンドボックスモード

**リンク:**
- リポジトリ: https://github.com/anthropics/claude-code
- CHANGELOG: https://github.com/anthropics/claude-code/blob/main/CHANGELOG.md

### Cline v3.36.1 (2025-11-07)

VS Code 拡張機能 Cline の最新バージョンがリリースされました。

**主な変更点:**
- **推論フィールドの削除**: AnthropicとGeminiプロバイダーから`reasoning_details`フィールドを削除し、互換性を向上
- **APPLY_PATCHツールのリファクタリング**: パッチ適用機能の信頼性が改善されました
- **プロバイダー情報の共有機能**: 異なるプロバイダー間での情報共有が改善
- **モデル選択の修正**:
  - AIHubMix でドロップダウンメニューからモデルを選択できない問題を修正
  - Grok-code を Grok-4 モデルファミリーチェックから除外
- **ネイティブツール呼び出しの改善**: 関数宣言ツールコンバーターの修正
- **GLMプロンプトの改善**: MCP ツール使用情報を GLM プロンプトに追加

**開発への影響:**
- 複数の AI プロバイダーとの統合がより安定
- パッチ適用機能の改善により、コード変更の適用がスムーズに
- モデル選択の柔軟性が向上

**リンク:**

### GitHub Copilot CLI v0.0.354 (2025-11-03)

GitHub Copilot CLI の最新版がリリースされました。

**主な変更点:**
- **エラーハンドリングの改善**: `-p`モードがLLMバックエンドエラー（認証失敗、クォータ枯渇、ネットワーク問題）で失敗した場合、非ゼロの終了コードを返すように
- **MCP サーバーツール通知のサポート**: MCP サーバーからのツール通知に対応
- **認証の柔軟性向上**: `COPILOT_GITHUB_TOKEN` 環境変数をサポート（`GH_TOKEN`より優先）
- **シェルコマンドの安全性向上**: heredocハンドリングの改善により、コマンド外でのより適切な処理を実現
- **UI改善**: Diffハンクラインがdiffボックスの幅を適切に埋めるように
- **GitHub Actions統合**: MCP サーバーが GitHub Actions 環境で `GITHUB_WORKSPACE` を自動的にワーキングディレクトリとして使用
- **コマンド修正**:
  - `/delegate` コマンドがローカル変更が存在しない場合でも正しく動作
  - ファイル名に特殊文字を含むカスタムエージェントが失敗しない
- **エラーメッセージの改善**: `/model` コマンドで非サポートモデルを使用した際のエラーメッセージが改善
- **代替プロバイダーのサポート**: 異なる OpenAI ベース URL 使用時に代替モデルプロバイダーが正しく動作

**実用的な影響:**
- CI/CD パイプラインでの信頼性が向上（GitHub Actions との統合改善）
- エラーハンドリングの改善により、スクリプトでの使用がより堅牢に
- 認証の柔軟性により、複数の環境での設定が容易に

**開発ワークフローへの影響:**
- ターミナルワークフローでの生産性が向上
- MCP サーバー統合により、外部ツールとの連携が強化
- GitHub Actions での自動化がよりスムーズに

**リンク:**
- リポジトリ: https://github.com/github/copilot-cli
- リリースノート: https://github.com/github/copilot-cli/releases/tag/v0.0.354
