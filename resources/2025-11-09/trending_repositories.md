# AIトレンドリポジトリ - 2025-11-09

## [Strix](https://github.com/usestrix/strix)

### 概要

セキュリティテストの世界に、新しい風が吹いています。Strix は、自律型AIエージェントを活用した動的セキュリティ評価フレームワークです。従来の静的解析ツールとは一線を画し、実際にコードをサンドボックス環境で実行しながら脆弱性を発見し、PoC（概念実証）を自動生成します。

開発者にとって画期的なのは、このツールが「AI の推論能力」と「実際の動的実行」を組み合わせている点です。単にコードをスキャンするだけでなく、HTTP プロキシを通じてリクエスト・レスポンスを分析し、ブラウザ自動化で複数タブのテストを行い、ターミナル環境でコマンドを実行し、Python ランタイムでカスタムエクスプロイトまで開発します。

### 主な機能

- **包括的な脆弱性検出**: アクセス制御の欠陥（IDOR、権限昇格）、インジェクション攻撃（SQL、NoSQL、コマンド）、サーバーサイド脆弱性（SSRF、XXE）、クライアントサイド問題（XSS）、認証の脆弱性まで幅広くカバー
- **Graph of Agents アーキテクチャ**: 専門化されたエージェントが並列で協調動作し、複数のターゲットに対してスケーラブルかつ協調的なセキュリティテストを実現
- **サンドボックス実行**: Docker コンテナ内で隔離されたテストを実行するため、本番システムに影響を与えずに安全にテスト可能
- **CI/CD統合**: GitHub Actions を通じて自動化でき、コード変更のたびに自動的にセキュリティ検証が実行される
- **複数LLMサポート**: 統一インターフェースで複数のLLMプロバイダーに対応

### 注目される理由

従来のペネトレーションテストは、誤検知が多かったり、手動作業に数週間かかったりと課題が山積みでした。Strix は、これらの問題を AI エージェントの自律性と動的実行の組み合わせで解決しようとしています。

特に注目すべきは、CI/CDパイプラインへの統合が容易な点です。セキュリティテストを「開発の最後に行う重い作業」から「コミットごとに自動実行される軽量なチェック」へと変革できる可能性を秘めています。Apache 2.0 ライセンスでオープンソース化されており、5.7k のスターを獲得していることからも、コミュニティからの期待の高さがうかがえます。

AI 駆動のセキュリティテストは、まだ発展途上の分野ですが、Strix のような実践的なツールの登場により、セキュアな開発がより身近になってきています。

## [Cognee](https://github.com/topoteretes/cognee)

### 概要

AI エージェントの「記憶」をたった6行のコードで実装できる、それが Cognee です。このオープンソースプラットフォームは、生データを AI エージェント向けの持続的で動的なメモリに変換します。単なるベクトル検索ではなく、グラフデータベースと組み合わせることで、ドキュメントを意味的に検索可能かつ関係性を持った形で管理できるのが特徴です。

従来の RAG（Retrieval-Augmented Generation）アプローチの限界を超えるため、Cognee はベクトル検索だけでなく知識グラフ推論を統合しています。これにより、AI は単に似た内容を探すだけでなく、情報間の関係性まで理解した上で回答できるようになります。

### 主な機能

- **ハイブリッド検索**: ベクトル検索とグラフデータベースを組み合わせ、意味的類似性と関係性マッピングの両方を活用
- **柔軟なデータパイプライン**: Pythonic ECL（Extract, Cognify, Load）パイプラインで30以上のデータソースから取り込み可能
- **2つのデプロイメントオプション**: セルフホスト版のオープンソース版と、UIダッシュボードと分析機能を備えたマネージドCognee Cloud
- **高度なカスタマイズ**: ユーザー定義タスク、モジュラーパイプライン、組み込み検索エンドポイント
- **エージェント記憶**: 自律エージェント向けの会話とコンテキストの永続化

### 注目される理由

AI エージェントの実用化において、「記憶」は最も重要な課題の一つです。会話履歴を保持するだけでは不十分で、過去の情報を適切に思い出し、関連する知識を結びつけて推論する能力が求められます。

Cognee は、この問題に対して実践的なソリューションを提供しています。pip でインストールし、LLM API キーを設定すれば、add（追加）、cognify（認知化）、memify（記憶化）、search（検索）、delete（削除）という5つのコア操作だけでエージェントに記憶機能を付与できます。

8.2k のスターと活発なコミュニティが示すように、エンタープライズグレードのエージェントメモリソリューションへの需要は高まっています。特に、長期的な文脈を保持しながら複雑なタスクを遂行する必要があるビジネスアプリケーションにおいて、Cognee のようなツールは不可欠になってくるでしょう。

知識グラフとベクトル検索の融合というアプローチは、次世代の AI エージェントアーキテクチャの基盤となる可能性を秘めています。
