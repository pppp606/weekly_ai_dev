## [Skyvern](https://github.com/Skyvern-AI/skyvern)

### 概要

ブラウザ自動化の世界に革命が起きています。Skyvern は、LLM とコンピュータビジョンを組み合わせて、ブラウザ操作を自動化するオープンソースフレームワークです。従来の自動化ツールが XPath や CSS セレクタに依存していたのに対し、Skyvern は Vision LLM を使って「人間のように」Web サイトを理解し、操作します。

開発者にとって画期的なのは、一度も見たことのない Web サイトに対しても、カスタムコードを書かずに自動化できる点です。レイアウトが変更されても動作し続けるため、従来の RPA（Robotic Process Automation）ツールで悩まされてきた「メンテナンス地獄」から解放されます。

### 主な機能

**コア機能**
- **ビジュアル要素認識**: 静的なセレクタに依存せず、視覚的に要素を識別するため、レイアウト変更に強い
- **未知のサイトへの対応**: 初めて訪れる Web サイトでもカスタムコードなしで操作可能
- **フォーム入力とデータ抽出**: 構造化スキーマでフォームを埋め、データを抽出
- **ファイルダウンロード**: 自動的にクラウドストレージへ統合

**高度な機能**
- **マルチタスクワークフロー**: ループ、条件分岐、データ変換を組み合わせた複雑な処理が可能
- **2FA サポート**: TOTP、メール、SMS による二要素認証に対応し、Bitwarden などのパスワードマネージャーとも連携
- **ライブストリーミングビューポート**: リアルタイムで実行状況を監視し、必要に応じて介入できる
- **自動化プラットフォーム統合**: Zapier、Make.com、N8N と連携して、より大規模なワークフローに組み込める
- **Model Context Protocol (MCP) サポート**: 柔軟に LLM を選択可能

### パフォーマンスと実績

Skyvern は WebBench 評価で **64.4% の精度**を達成し、特に WRITE タスク（フォーム入力、ダウンロード、認証）において**クラス最高のパフォーマンス**を記録しています。複雑な推論を必要とするタスクでも、異なる情報を総合して判断できる能力を持っています。

### 実用例

開発者が実際に活用できるシナリオは多岐にわたります：

- **請求書の一括ダウンロード**: 複数のベンダープラットフォームから自動的に請求書を取得
- **求人応募の自動化**: 複数の求人サイトへの応募プロセスを効率化
- **製造業の資材調達**: 部品や資材の発注プロセスを自動化
- **競合価格モニタリング**: ライバル企業の価格情報を定期的に収集
- **レガシー RPA の置き換え**: 既存の脆弱な自動化スクリプトを堅牢なソリューションに移行

### 注目される理由

従来のブラウザ自動化ツール（Selenium、Puppeteer など）は、DOM 構造に強く依存しているため、Web サイトのデザイン変更のたびにスクリプトを修正する必要がありました。これは開発者にとって大きな負担であり、特に複数のサイトを対象とする場合、メンテナンスコストが膨大になります。

Skyvern は、この問題を根本から解決します。Vision LLM を使って画面を「視覚的に理解」するため、HTML の内部構造が変わっても動作し続けます。これは、人間がブラウザを操作するときと同じアプローチです。ボタンがどこにあるかは、XPath ではなく「見た目」で判断します。

さらに、開発時間を**従来の RPA と比較して 10〜100 倍削減**できるという実績も注目に値します。カスタムコードを書く必要がないため、非エンジニアでも使いやすく、技術チームの負担を大幅に軽減します。

18.1k のスターを獲得し、活発な Discord コミュニティが形成されていることからも、実用性の高さがうかがえます。AGPL-3.0 ライセンスでオープンソース化されており、セルフホストも可能ですが、app.skyvern.com でクラウド版も提供されているため、インフラ管理なしで即座に利用開始できます。

「エージェントの群れが Web サイトを理解し、計画を立て、行動を実行する」という独自のアーキテクチャは、今後のブラウザ自動化のスタンダードになる可能性を秘めています。

## [LocalAI](https://github.com/mudler/LocalAI)

### 概要

クラウド API に頼らず、自分のマシンで AI を動かしたい。そんな開発者の願いを叶えるのが LocalAI です。このオープンソースプロジェクトは、OpenAI、Claude、その他の AI サービスに代わる**無料のローカル実行可能な代替手段**として、38.5k のスターを獲得しています。

最大の特徴は、OpenAI API との完全な互換性を持ちながら、GPU なしの消費者向けハードウェアでも動作する点です。既存のアプリケーションコードを一切変更せず、API エンドポイントを切り替えるだけで、ローカル AI に移行できます。

### 主な機能

**包括的な AI 機能**
- **テキスト生成**: llama.cpp、transformers、vLLM など複数のバックエンドをサポート
- **音声処理**: テキスト読み上げ（TTS）と音声認識（STT）
- **画像生成**: ローカルでの画像生成と Vision API
- **音声文字起こし**: 音声ファイルの自動文字起こし
- **埋め込み生成**: ベクトルデータベース用の埋め込みベクトル作成
- **関数呼び出し**: OpenAI 互換の Function Calling をサポート
- **オブジェクト検出とリランキング**: 高度な AI タスクにも対応

**インフラストラクチャの先進性**
- **P2P 分散推論**: 複数のマシンで推論を分散し、フェデレーテッドモードで動作可能
- **Model Context Protocol (MCP) サポート**: エージェント機能を統合
- **統合 WebUI**: チャット、画像生成、音声インターフェースを備えた使いやすい UI
- **バックエンドギャラリーシステム**: 自動 GPU 検出機能
- **Ollama と HuggingFace 統合**: OCI レジストリモデルと HuggingFace モデルをシームレスに利用

### ハードウェアアクセラレーション対応

LocalAI は、あらゆるプラットフォームで最適なパフォーマンスを実現します：

- **NVIDIA**: CUDA 11/12 をサポート
- **AMD**: ROCm 対応
- **Intel**: oneAPI でアクセラレーション
- **Apple**: Metal（M1/M2/M3+）による高速化
- **クロスプラットフォーム**: Vulkan、Jetson (ARM64) にも対応

### 最近のアップデート

LocalAI は活発に進化を続けています：

- **2025 年 10 月**: Model Context Protocol サポートを追加し、エージェント機能を強化
- **2025 年 8 月**: Mac チップ向けに MLX、MLX-VLM サポートを導入
- **2025 年 7 月**: 大規模なリファクタリングでバックエンドをメインバイナリから分離し、フットプリントを大幅に削減
- **2025 年 5 月**: 音声入力とリランキング機能、Realtime API を導入

### 実用例

開発者が LocalAI を選ぶ理由は、実用性の高さにあります：

- **プライベート・オンプレミス AI デプロイメント**: 機密データを外部に送信せず、社内で AI を活用
- **エッジコンピューティング**: ネットワーク接続が限られた環境や組み込みシステムでの AI 実行
- **プライバシー重視のアプリケーション**: ユーザーデータを保護しながら AI 機能を提供
- **カスタムモデルホスティング**: 独自にファインチューニングしたモデルを運用
- **フェデレーテッド AI**: 分散環境での協調的な AI 推論

### 注目される理由

AI 開発において、コストとプライバシーは常に大きな課題です。クラウド API は便利ですが、従量課金のコストが積み上がり、機密情報を外部に送信するリスクがあります。LocalAI は、これらの問題を一挙に解決します。

特に注目すべきは、**GPU なしでも動作する**という点です。多くの AI ツールが高価な GPU を要求する中、LocalAI は消費者向けのハードウェアで十分に動作します。もちろん、GPU があればさらに高速化しますが、必須ではありません。この「アクセシビリティ」が、LocalAI の大きな強みです。

また、OpenAI API との完全互換性により、既存のアプリケーションを**一切変更せずに**移行できます。`OPENAI_API_BASE` 環境変数を変更するだけで、ローカル実行に切り替えられます。これは、プロトタイプ開発時はクラウド API を使い、本番環境ではローカルに切り替える、といった柔軟な運用を可能にします。

P2P 分散推論やフェデレーテッドモードといった先進的な機能も、エンタープライズグレードの展開を視野に入れた設計です。複数のマシンで AI 処理を分散させることで、単一マシンの限界を超えたスケーラビリティを実現できます。

Model Context Protocol のサポートにより、エージェント機能との統合も容易になりました。これは、AI エージェントのローカル実行という新しいユースケースを開拓するものです。

38.5k のスターと活発なコミュニティ（Discord、Telegram）が示すように、LocalAI は「AI をローカルで動かす」というムーブメントの中心的存在となっています。プライバシー、コスト削減、データ主権を重視する開発者にとって、LocalAI は最も実用的な選択肢の一つです。
