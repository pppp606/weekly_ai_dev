# AI開発ツール リリース情報 (2026-01-11 〜 2026-01-18)

## 1. Claude Code

**バージョン**: 2.1.12 (最新) / 2.1.10, 2.1.11 他
**更新日**: 2026-01-17
**リポジトリ**: https://github.com/anthropics/claude-code
**CHANGELOG**: https://github.com/anthropics/claude-code/blob/main/CHANGELOG.md

### 主な変更点

#### 新機能 (2.1.10)
- **Setupフックイベント**: `--init`、`--init-only`、`--maintenance` CLIフラグで起動可能な新しいフックイベント。リポジトリのセットアップやメンテナンス操作に使用
- **OAuth URLコピーショートカット**: ログイン時にブラウザが自動で開かない場合、'c'キーでOAuth URLをコピー可能に
- **プラグイン表示改善 (VSCode)**: プラグイン一覧にインストール数を表示、プラグインインストール時に信頼性警告を追加

#### 修正 (2.1.10-2.1.12)
- heredoc内のJavaScriptテンプレートリテラル（`${index + 1}`など）を含むbashコマンド実行時のクラッシュを修正
- REPLが完全に準備完了する前に入力されたキーストロークをキャプチャするように起動処理を改善
- ファイル候補を挿入テキストではなく削除可能な添付ファイルとして表示するように改善
- MCP HTTP/SSEトランスポートでの過剰な接続リクエストを修正
- メッセージレンダリングバグを修正

#### 重要な設定 (2.1.9)
- **`auto:N`構文**: MCP tool search自動有効化のしきい値をコンテキストウィンドウのパーセンテージ(0-100)で設定可能
- **`plansDirectory`設定**: プランファイルの保存場所をカスタマイズ可能
- **`${CLAUDE_SESSION_ID}`**: スキルからセッションIDにアクセスするための文字列置換をサポート

#### 開発者への影響
- Setupフックを活用することで、プロジェクト固有の初期化やメンテナンスタスクを自動化できる
- MCP接続の安定性が向上し、長時間のセッションでの信頼性が改善

- MCPサーバーがカスタムエージェントで正しく起動するように修正

- カスタムエージェント検出がシンボリックリンクを追跡
- スクリーンリーダーモードでログイン中にスピナーの代わりに静的テキストを表示
- 推論ビュー設定がセッション間で保持
- CLIで作成されたファイルが@メンション可能に

#### 実用的な使用例
```bash
# バックグラウンドでタスクを実行

/cd /path/to/project
```

### 主な変更点 (v0.24.0)

#### Agent Skills - 重要な新機能
- **Agent Skills基盤**: コアスキルインフラストラクチャと階層型ディスカバリを実装
- **自律アクティベーションツール**: スキルの自律的なアクティベーションとコンテキスト注入を実装
- **エージェント統合**: システムプロンプト認識を備えたエージェント統合を実装
- **拡張機能サポート**: 拡張機能サポートとセキュリティ開示を追加
- **`gemini skills`コマンド**: スキル管理用のCLIコマンドを追加
- **スキルリロード**: `/skills reload`コマンドを実装

#### リモートエージェント機能
- **リモートエージェントサポート**: リモートエージェントとマルチエージェントTOMLファイルのサポートを追加
- **リモートエージェント登録**: エージェントレジストリにリモートエージェントを追加

#### セキュリティ強化
- **フォルダー信頼サポート**: フックにフォルダー信頼サポートを追加
- **プロジェクトレベルフック警告**: セキュリティ警告を追加

- **STOP_EXECUTION実装**: フックでの粒度の高い停止およびブロック動作を実装
- **hooks.enabled設定**: フックの有効/無効を設定可能に
- **フック実行の視覚的インジケーター**: フック実行時の視覚的表示を追加

#### その他の改善
- **モデル永続化**: `/model`コマンドでの自動モデル永続化
- **動的モード対応ポリシー評価**: ポリシーエンジンで動的なモード認識評価を実装
- **ツール入力変更サポート**: フックからのツール入力変更をサポート
- **リモート管理設定**: 管理者用リモート設定と`secureModeEnabled`/`mcpEnabled`を導入

#### 使用例
```bash

gemini skills

# スキルのリロード
/skills reload

# モデルの永続化
/model [model-name]
```

## 6. Cursor

**バージョン**: CLI Jan 16, 2026
**更新日**: 2026-01-16
**ウェブサイト**: https://cursor.com
**Changelog**: https://cursor.com/changelog

### 主な変更点

#### 新機能
- **Plan Mode in CLI**: `/plan`または`--mode=plan`でコーディング前に設計アプローチを検討、明確化のための質問を生成
- **Ask Mode in CLI**: `/ask`または`--mode=ask`で変更を加えずにコードを探索
- **Cloud Handoff**: メッセージの先頭に`&`を付けてCloud Agentsにプッシュ、Web/モバイルで再開可能
- **Word-level Inline Diffs**: CLI出力で変更箇所を正確にハイライト表示
- **One-click MCP Authentication**: 自動コールバック処理で認証済みMCPへの即座のエージェントアクセスを実現、`/mcp list`でインタラクティブ管理

#### 実用的な使用例
```bash
# プランモードで設計を検討
cursor --mode=plan

# 質問モードでコード探索
cursor /ask "この関数の目的は?"

# Cloud Agentsにタスクを委譲
& このコードをリファクタリングして
```

## 7. Kiro

**バージョン**: CLI v1.24.0
**更新日**: 2026-01-16
**ウェブサイト**: https://kiro.dev
**Changelog**: https://kiro.dev/changelog/

### 主な変更点

#### 新機能
- **Skills for Progressive Context Loading**: 大規模なドキュメントセット向けの新しいリソースタイプ。起動時にメタデータをロードし、オンデマンドで完全なコンテンツを取得
- **Custom Diff Tools**: delta や difftastic などの外部diffツールを設定可能に（組み込みインラインdiffの代わりに使用）
- **AST Pattern Tools**: テキスト正規表現ではなく構文ツリーパターンを使用したコード変換のための新しいpattern-searchとpattern-rewrite機能
- **Enhanced Code Intelligence**: 18言語でLSPセットアップ不要のコード理解を実現
- **Conversation Compaction**: `/compact`コマンドでコンテキスト制限に近づいた際に会話履歴を要約してコンテキストスペースを管理
- **Granular URL Permissions**: エージェント設定で正規表現パターンを使用してエージェントがアクセスできるURLを制御
- **Remote Authentication**: SSH、SSM、コンテナでのリモートマシンでのGoogleおよびGitHubサインインをサポート

#### その他の改善
- ナレッジベースのインデックスがセッション間で保持される
- キーボードショートカットでオプションのウェルカムメッセージ付きのクイックエージェント切り替えが可能

**リポジトリ**: https://github.com/microsoft/vscode

### AI/Copilot関連の変更点

- キーボードアクセスのサポート
- 状態と経過時間によるセッショングループ化
- 変更されたファイルと関連するプルリクエストの表示

- **セッションベースのピッカー**: チャットセッション用のQuick Pickで以前の会話を表示、アーカイブ・リネーム・削除が可能

#### Terminal Tool Shell History
- シェル統合が有効な場合、ターミナルツールで実行されたコマンドがシェル履歴に表示されない（bash、zsh、pwsh、fish対応）
