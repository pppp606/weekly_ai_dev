## 1. Google Gemini CLI v0.27.0〜v0.27.3

- **リリース日**: 2026-02-04（v0.27.0）、2026-02-05（v0.27.2）、2026-02-06（v0.27.3）
- **リポジトリ**: https://github.com/google-gemini/gemini-cli
- **リリースページ**: https://github.com/google-gemini/gemini-cli/releases/tag/v0.27.3

### 主な変更点

#### 新機能
- **イベント駆動型ツール実行スケジューラ**: ツール実行をイベント駆動方式に移行し、デフォルトで有効化。非インタラクティブモードやサブエージェントにも適用され、並列実行の効率が向上
- **Hooks システムのデフォルト有効化**: `beforeTool`/`afterTool` フックが新しいフックシステムに移行され、デフォルトで有効に。フック注入コンテキストは別のXMLタグでラップされセキュリティが強化
- **Plan Mode**: `Shift+Tab` でのモード切替に Plan Mode が追加。承認モード設定の永続化、AskUser ツールによる質問機能、永続的なプランファイル保存に対応
- **`/rewind` コマンド**: 会話の巻き戻し機能を実装
- **`/agents` コマンド**: エージェント設定の管理コマンドを追加。プロジェクトレベルのサブエージェントの初回実行体験も改善
- **AskUser ツール**: エージェントからユーザーへ質問を行うためのツールを追加（スキーマ定義、UI、ダイアログ連携）
- **Agent Skills のデフォルト有効化**: Skills 設定がstableに昇格
- **MCP サーバーの有効/無効切替**: 個別の MCP サーバーを有効/無効にするコマンドを追加
- **画像ペースト対応（Linux）**: Wayland と X11 での画像ペースト機能を追加
- **大規模テキストペーストのプレースホルダー表示**: 大きなテキストペーストを `[Pasted Text: X lines]` プレースホルダーに変換し、ダブルクリックで展開/折りたたみ可能
- **ツール承認のキュー UX**: ツール確認をキュー形式の UX に変更
- **Ctrl-O による承認の展開/スクロール**: alternate buffer mode での承認表示を改善
- **JetBrains IDE 検出**: JetBrains IDE の検出機能を追加
- **JSON レンダリング**: ツール出力のJSON表示を整形表示に改善
- **`gemini extensions list --output-format=json`**: 拡張機能一覧のJSON出力をサポート

#### 改善
- **動的変数置換**: システムプロンプトオーバーライドで動的変数置換をサポート
- **モデルファミリー固有のシステムプロンプト**: Gemini 3 向けの数値複雑度スコアリングの A/B テストを導入
- **非インタラクティブモードの改善**: デフォルトでインタラクティブモードに切替（位置引数使用時）
- **ripgrep 出力のストリーミング化**: OOM を防止するために grep/ripgrep 出力をストリーム化
- **サブエージェント委譲のリファクタリング**: エージェントごとに1ツールに変更

#### バグ修正
- 認証初回失敗時のクラッシュ防止、ANSI エスケープシーケンスのサニタイズ、ターミナルリサイズ時のレンダリング安定化、ディスクフル時のチャット記録エラーハンドリングなど多数の修正

#### v2.1.34

#### v2.1.36
- **Fast Mode の Opus 4.6 対応**: Opus 4.6 で Fast Mode が利用可能に（詳細: https://code.claude.com/docs/en/fast-mode ）

#### v2.1.37
- `/fast` 有効化後の即時利用に関する問題を修正

### 詳細: エージェントチーム機能

マルチエージェント協調を実現する研究プレビュー機能。複数のClaude エージェントがチームとして協力し、タスクを分担して並列処理可能。

**使用方法**:
```bash
# 環境変数の設定が必要
export CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1
```

**関連フックイベント**:
- `TeammateIdle`: チームメイトがアイドル状態になった際に発火
- `TaskCompleted`: タスク完了時に発火

**サブエージェント制御**:
```yaml
# エージェント定義のフロントマターで制限可能
tools:
  - Task(code_reviewer)
  - Task(test_writer)
```

### 詳細: メモリ機能

エージェントに永続メモリを付与し、セッション間で学習内容を保持。

**フロントマター設定例**:
```yaml
memory: user    # ユーザースコープ（全プロジェクト共通）
memory: project # プロジェクトスコープ
memory: local   # ローカルスコープ
```

#### エージェントセッション管理
- **マルチ環境エージェントセッション**: ローカル、バックグラウンド、クラウドの各環境で複数のエージェントセッションを並列実行可能。統合ビューから切替可能
- **サブエージェントの独立コンテキスト**: サブエージェントは独自のコンテキストウィンドウで動作し、メインエージェントのコンテキストを消費しない。独立タスクの並列実行により高速化
- **Search サブエージェント（実験的）**: コードベースの反復検索を行う独立エージェントループ
- **クラウドエージェント強化**: モデル選択、サードパーティコーディングエージェント（Claude、Codex）のプレビュー対応、GitHub リポジトリからのカスタムエージェント選択

#### エージェントカスタマイズ
- **Agent Skills（GA）**: 正式リリース。`.github/skills`、`.claude/skills` などからスキルを検出・読み込み。「Chat: Configure Skills」「Chat: New Skill File」コマンドで管理
- **`/init` コマンド**: ワークスペース指示書を自動生成（AIコンベンション検出、プロジェクト構造分析）
- **組織全体の指示**: GitHub 組織がカスタム指示を設定し、チーム全体のチャットセッションに自動適用
- **カスタムエージェント呼び出し制御**: `user-invokable`、`disable-model-invocation`、`agents` フロントマター属性を追加
- **複数モデルサポート**: カスタムエージェントが複数モデルを指定可能（フォールバック対応）

#### エージェント拡張性
- **エージェントオーケストレーション**: 複数エージェントパターンで作業を専門エージェントに分散。コンテキスト効率、専門性、並列実行が利点
- **Claude Agent（Preview）**: Anthropic Claude Agent SDK の公式統合。GitHub Copilot サブスクリプションに含まれる Claude モデルへの委譲が可能
- **Anthropic Messages API**: インターリーブドシンキング対応、思考予算設定（`github.copilot.chat.anthropic.thinking.budgetTokens`）
- **MCP Apps サポート**: MCP Apps からのリッチでインタラクティブな UI を自動レンダリング

#### エージェント最適化
- **Copilot Memory（Preview）**: セッション間で情報を保存・呼び出すメモリシステム。`github.copilot.chat.copilotMemory.enabled` で有効化
- **外部インデックス対応（Preview）**: GitHub 以外のワークスペースもリモートインデックスで高速コード検索が可能

#### エージェントセキュリティ & 信頼性
- **ターミナルサンドボックス（実験的）**: ファイルシステムアクセスをワークスペースフォルダに制限、ネットワークアクセスを信頼済みドメインに制限（macOS/Linux 対応）
- **ターミナル自動承認の拡充**: 安全なコマンド（ディレクトリ操作、Docker サブコマンド等）の自動承認

#### Chat UX
- **思考トークンの表示**: Anthropic の思考トークンをカスタマイズ可能なスタイルで表示
- **Mermaid ダイアグラムレンダリング**: パン/ズーム操作、フルエディタ表示、ソースコードコピー機能
- **コンテキストウィンドウインジケータ**: チャット入力エリアにトークン使用量をカテゴリ別に表示
